# Setting Up

Follow these steps to run the project in your local environment and start contributing.

## Prerequisite

All of these must be done before anything:

- [Install git](https://git-scm.com/install/windows).
- [Install Composer](https://getcomposer.org/download/).
- [Install Node.js](https://nodejs.org/en/download).

## Local web server setup

You may use Laragon, XAMPP, or anything you like.

### Laragon

1. [Install laragon](https://laragon.org/download).

1. Run the `laragon.exe`.
1. Click on `Start All`.
1. [Clone the repository](#cloning-repository) inside the `../www` folder.
1. Do every [project setup](#project-setup) step from start to finish.
1. Open `http://{project_name}.test/` e.g. http://project-manager.test/

### XAMPP

1. [Install XAMPP](https://www.apachefriends.org/).

1. Navigate to `../apache/conf/extra` and edit the `httpd-vhosts.conf` file with a text editor, then add the code snippet below with your own path:
    ```markup
    <VirtualHost *:80>
        # highlight-start
        DocumentRoot "C:/XAMPP/htdocs/tournament-manager/public"
        # highlight-end
        ServerName http://tournament-manager.test/
        # highlight-start
        <Directory "C:/XAMPP/htdocs/tournament-manager/public">
        # highlight-end
            AllowOverride All
            Require all granted
        </Directory>
    </VirtualHost>
    ```
1. Navigate to `C:/Windows/System32/drivers/etc/` and open the `hosts` file as administrator, then add this line:
    ```markdown
    127.0.0.1 tournament-manager.test
    ```
    <details>
    <summary>How to open `hosts` file as administrator</summary>

    If the button to `Run as administrator` did not appear, do these steps:
    1. Open your text editor as administrator.
    1. Click on `File -> Open`.
    1. Navigate to the `C:/Windows/System32/drivers/etc/`.
    1. Open `hosts`.
    </details>
1. Run the `XAMPP Control Panel` as administrator.
    :::tip
    You can set the shortcut to always run as administrator by <br />
    right clicking the shortcut -> Properties -> Advanced -> check Run as administrator.
    :::
1. Start both `Apache` and `MySQL`. These can be set to autostart in Config.
1. [Clone the repository](#cloning-repository) inside the `../htdocs` folder.
1. Do every [project setup](#project-setup) step from start to finish.
1. Open http://tournament-manager.test

## Project setup

### Cloning repository

Run these commands in a terminal of your choice.

```bash
git clone https://github.com/Xerillia/tournament-manager.git
cd tournament-manager
```

### Installing dependencies

```bash
composer install
npm install
```
:::note
Ensure you are in the project folder, e.g. `../tournament-manager` before running the commands.
:::

### Creating `.env`

The following section is done inside the project folder.

1. Duplicate the `.env.example` file in the project folder and rename it to `.env`

1. Generate an `APP_KEY` by running this command:
    ```bash
    php artisan key:Generate
    ```
1. Set the `APP_URL` field accordingly.
1. Set the `DB_CONNECTION` field and its configuration.
    :::danger
    Do not use `sqlite` because it will not work.
    :::
1. Setup the OAuth APIs by following the sections below:

### OAuth setups

#### osu!

1. Open the [osu! account settings](https://osu.ppy.sh/home/account/edit) page and scroll down to the `OAuth` section.

1. Click on `New OAuth Application`.
1. Set the `Application Name` to anything you want.
1. Set the `Application Callback URLs` to `{APP_URL}/osu/callback`, e.g. http://tournament-manager.test/osu/callback.
    :::info
    The URI path must be `/osu/callback`.
    :::
1. Click on `Register application`.
1. Click on `Edit` from the newly made application.
1. Copy the value under `Client ID` and paste it to the `OSU_CLIENT_ID` field in the `.env` file.
1. Click on `Show client secret`, then copy the value under `Client Secret`, and paste it to the `OSU_CLIENT_SECRET` field.
1. Done.

#### Discord

1. Open the [Discord Developer Portal](https://discord.com/developers/applications) page.

1. Click on `New Application`.
1. Set the `Name` to anything you want, then create the app.
1. Navigate to the `OAuth2` tab through the sidebar on the left.
1. Click on `Add Redirect` then set it to `{APP_URL}/discord/callback`, e.g. http://tournament-manager.test/discord/callback.
    :::info
    The URI path must be `/discord/callback`.
    :::
1. Copy the `Client ID` value and paste it to the `DISCORD_CLIENT_ID` field in the `.env` file.
1. Click on `Reset Secret` to generate a new `Client Secret`, then copy the value and paste it to the `DISCORD_CLIENT_SECRET` field.
    :::warning
    The value will be shown **only once**, then be permanently hidden once the page is closed or refreshed.
    :::
1. Done.

### Database migrations

Run this command in the project directory to load the database tables.

```bash
php artisan migrate
```

**Optional:** seed the tables with predetermined and/or generated data.

```bash
php artisan db:seed
```

### Development environment

**Always** have this command running while developing the app:

```bash
composer run dev
```

## Building application

```bash
npm run build
```
